# Catálogo de blocos (guia prático)

Use este arquivo como mapa de intenção para escolher blocos `@shadcnblocks` por objetivo de tela.

> O catálogo oficial muda com o tempo. Sempre valide via `npx shadcn@latest search @shadcnblocks -q "<termo>"`.

## Landing pages

- Hero, CTA, feature grid, social proof, pricing, FAQ, footer.
- Busca sugerida:
  - `hero`
  - `pricing`
  - `faq`
  - `testimonial`

## SaaS e dashboard

- Sidebars, KPIs, cards de métricas, tabelas, filtros e cabeçalhos de app.
- Busca sugerida:
  - `dashboard`
  - `analytics`
  - `table`
  - `sidebar`

## Ecommerce

- Product grid, cards de produto, carrinho, checkout, reviews.
- Busca sugerida:
  - `product`
  - `cart`
  - `checkout`
  - `store`

## Auth e onboarding

- Login, signup, reset, passos guiados e telas de boas-vindas.
- Busca sugerida:
  - `login`
  - `signup`
  - `auth`
  - `onboarding`

## Conteúdo e marketing

- Blog listing, detalhe de post, seções institucionais e contato.
- Busca sugerida:
  - `blog`
  - `article`
  - `contact`
  - `about`

## Fluxo recomendado para seleção

1. Buscar 5 a 10 candidatos por intenção.
2. Inspecionar os finalistas com `view`.
3. Escolher o item com menor custo de adaptação.
4. Instalar e ajustar para tokens/ícones/aliases do projeto.
